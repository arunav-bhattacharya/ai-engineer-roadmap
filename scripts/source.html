How to study — read once, refer back

Budget ~10 hrs/week: ~2 hrs read/watch, ~6 hrs coding, ~2 hrs writing (a README or short post — this is where it snaps into your head). The single biggest predictor of retention is the build-to-theory ratio.

70% BUILD30% READ

The pattern that works for engineers like you:

  • Read the topic intro (15 min) — skim, don't deep-read.
  • Watch one video on 1.5x.
  • Build the smallest thing that demonstrates the concept, with no framework. Highest-leverage step.
  • Rebuild with the framework — now you see what it was hiding.
  • Write a 200-word note: "what I'd use this for at work."

Part I vs Part II: Part I is hands-keyboard build cadence to a shippable capstone — don't stall on theory. Part II is the depth that makes you senior in AI, not just functional: it's anchored on Chip Huyen's AI Engineering (O'Reilly) — keep the book open as the spine for W11–16.

Don't fall for: tutorial hell, framework-collecting, chasing every model release. Pick one frontier API (Anthropic) + one open-source path (HuggingFace + Ollama) and stick to them.

Day-0 setup — do before Week 0

  • Editor: VS Code + Python/Jupyter extensions, or PyCharm if you want JetBrains-familiar.
  • Package manager: uv (fast, lockfile-based, Gradle-like).
  • API keys: Anthropic ($5 free), OpenAI ($5), Groq (free tier, fast inference for experiments).
  • Local models: install Ollama, pull llama3.2:3b and qwen2.5:7b.
  • Vector DB: start with Chroma (in-process, zero-ops). Move to pgvector/Qdrant later.
  • Observability: sign up for Langfuse (open-source, self-hostable).
  • Git: one repo ai-learning-lab, one folder per week.
  • Claude Code: install Claude Code now — Week 1 turns it into your daily driver.

WEEK 09–10

Capstone — Billpay Ops Copilot

~20 hrs

Goal: ship one real, end-to-end agentic system you'd demo to a hiring manager. Uses everything: structured outputs, RAG, multi-framework agents, MCP, evals, Langfuse, and Temporal orchestration. No tutorials — just build. Build it JVM-native — Quarkus + Quarkus-LangChain4j + the Temporal Java SDK, reusing your Week 2B work. The Python weeks were for learning; the capstone ships on your stack, which makes it a far stronger portfolio piece for your background.

The build — recommended pick (strongest for your portfolio)

An agent that diagnoses a stuck Temporal workflow given a workflow ID, then proposes (and optionally executes, behind a human-approval signal) a mitigation. It sits exactly at the intersection of your moat and AI engineering.

  • Input: a workflow ID (or an alert webhook).
  • Tools / MCP servers: query Temporal API (history, pending activities, stuck task queues); read Oracle business state via your Week-5 read-only MCP server; RAG over runbooks + Temporal docs (Week 3); a metrics tool (shard latency, sync-match rate).
  • Reasoning: a Quarkus-LangChain4j agent (or LangGraph if you'd rather stay in Python) classifies the failure (sticky-queue starvation, poller starvation, IO:XactSync contention, prepared-statement pinning, …) and drafts a mitigation.
  • Orchestration: the whole diagnosis runs as a Temporal workflow (Java SDK) — durable, retryable, with a human-approval Signal before any write action.
  • Quality gates: eval suite (Week 7) with golden incidents; LLM-as-judge on diagnosis correctness.
  • Observability: full Langfuse tracing of every tool call and token.

Alternative picks (if you want a change of domain)

  • Code Review Agent — reviews Kotlin PRs against your team style guide + anti-patterns; RAG over the guide + history.
  • India Relocation Concierge — multi-step planning (schools, neighborhoods, doc checklists). Personally useful, less technically flashy.
  • Wildlife Photography Assistant — RAG over EXIF + field guides; suggests lens/settings. Niche but distinctive.

Pitfalls specific to senior backend engineers (you)

  • You now own overlapping courses — don't binge them linearly. Use the resource map: one primary per week, the rest as reference. Watching all six Ed Donner courses + ByteByteGo + Cohen end-to-end is hundreds of hours and mostly redundant.
  • Don't let Part II block shipping. The capstone (W10) is the hireable artifact; Weeks 11–17 deepen you but a portfolio project beats half-read theory. Ship first.
  • Don't chase "the correct framework." Week 5 is breadth on purpose — learn all four, commit to none.
  • Your backend rigor is a superpower past Week 4. Most AI engineers can't reason about throughput, idempotency, retries, or transactional semantics — your capstone and Week 15 should be 10× more production-ready than the median learner's.
  • Don't fine-tune unless evals say to. The #1 trap for technical learners.
  • Your moats are Weeks 2B, 8, and 15. JVM-native AI + Temporal orchestration + inference optimization. Build the capstone on the JVM and write the blog posts — they're your differentiated interview assets.
